## 📝 1️⃣ README.md — פתיח מלא למאגר

**Prompt:**

> Create a full README.md for a new GitHub repository following RepoCraft standards.
> Include: Title, badges, description, features, quick install, usage, roadmap, contribution rules, license note, links and a centered hero-banner section.
> Formatting: Markdown + emoji + sections divided with headers.
> Keep tone professional & exciting.

---

## 🛡️ 2️⃣ LICENSE — MIT / Apache / GPL

**Prompt:**

> Add a license file to the repo in plain text.
> Choose from MIT / Apache-2.0 / GPL-3.0 based on open-source standards.
> Ensure the owner name is “AnLoMinus – Leon Yaakobov” and include the current year.
> File name: LICENSE (no extensions).

---

## 🤝 3️⃣ CONTRIBUTING.md — מדיניות תרומות

**Prompt:**

> Create a CONTRIBUTING guide tailored for a creative + technical project.
> Include: prerequisites, branching strategy, commit message rules, PR flow, review style, feedback etiquette, and label usage.
> Write short, clear bullets + supportive tone.

---

## 🧘 4️⃣ CODE_OF_CONDUCT.md — קהילה מכבדת

**Prompt:**

> Generate a Code of Conduct file based on the Contributor Covenant v2.1.
> Keep messages: respect, inclusion, zero harassment, good vibes.
> Add report-contact: “[contact@anlominus.com](mailto:contact@anlominus.com)”.
> Markdown headings and bullet points required.

---

## 🔐 5️⃣ SECURITY.md — אבטחה ודיווח פרצות

**Prompt:**

> Create a SECURITY policy for reporting vulnerabilities.
> Include: responsible disclosure rules, forbidden exploit activities, expected response time, and a PGP note (placeholder).
> Email for disclosure: [security@anlominus.com](mailto:security@anlominus.com).

---

## 🧩 6️⃣ CHANGELOG.md — היסטוריית גרסאות

**Prompt:**

> Generate a CHANGELOG based on Keep a Changelog format.
> Sections: [Unreleased], Added, Changed, Fixed, Security.
> Each future release follows SemVer (v0.1.0 etc.).

---

## 🧪 7️⃣ ISSUE TEMPLATES — באגים & פיצ׳רים

**Prompt:**

> Create two Issue templates:
> (A) Bug Report — steps to reproduce, expected vs. actual, environment
> (B) Feature Request — user story, acceptance criteria, alternatives
> Format in YAML for `.github/ISSUE_TEMPLATE/`.

---

## 🔄 8️⃣ PR_TEMPLATE.md — תבנית Pull Request

**Prompt:**

> Write a PR template with:
> Summary, Motivation, Changes list, Screenshots, Tests done, Checklist (lint, docs updated, no breaking changes).
> Tone: supportive, constructive, proud of work.
